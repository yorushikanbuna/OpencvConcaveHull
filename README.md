# 凹包生成器 (Concave Hull Generator)

从POS文件读取影像位置坐标，提取XY平面点集，生成凹包（Concave Hull）边界，输出文本文件和可视化图像。

## 项目结构

```
ConcaveHullProj/
├── main.cpp              # 程序入口，流程编排
├── pos_reader.h/cpp      # POS文件解析模块
├── concave_hull.h/cpp    # 凹包计算核心算法
├── output.h/cpp          # 文本输出 + 图像可视化
├── tests.h/cpp           # 单元自测
├── CMakeLists.txt        # CMake构建配置
├── vcpkg.json            # vcpkg依赖清单
└── README.md
```

## 快速开始

### 1. 前置条件

- 安装 [Visual Studio 2019+](https://visualstudio.microsoft.com/)（含MSVC工具链）
- 安装 [vcpkg](https://github.com/microsoft/vcpkg) 并设置环境变量

### 2. 构建

```bash
cd ConcaveHullProj
mkdir build && cd build
cmake .. -DCMAKE_TOOLCHAIN_FILE=<vcpkg路径>/scripts/buildsystems/vcpkg.cmake
cmake --build . --config Release
```

### 3. 运行

```bash
# 使用自动确定的k值
./Release/concave_hull.exe

# 手动指定k值（k越小凹包越紧密）
./Release/concave_hull.exe 30
```

### 4. 输出

程序会在POS文件所在目录生成两个文件：

- `concave_hull.txt` — 凹包边界顶点坐标（每行 `x y`）
- `concave_hull.png` — 可视化图像（蓝点=原始点集，红线=凹包边界，绿点=边界顶点）

## POS文件格式

```
A/A0001.JPG 528862.590974 3802421.315441 77.219743
A/A0002.JPG 528862.590974 3802421.315441 77.216743
...
```

每行由空格分隔，四列分别为：**影像名 X坐标 Y坐标 Z坐标**。程序仅使用X和Y坐标，Z坐标被忽略。空行和以 `#` 开头的注释行会自动跳过，重复坐标自动去重。

## k参数说明

`k` 是控制凹包松紧程度的参数，取值范围建议 **1 ~ 200**：

| k值 | 效果 | 适用场景 |
|-----|------|----------|
| 3 ~ 10 | 紧密包裹，严格贴合外轮廓 | 建筑轮廓提取、精确边界 |
| 11 ~ 30 | 适度包裹，平滑细节 | 一般场景、默认推荐 |
| 31 ~ 60 | 宽松包裹，趋向凸包 | 区域概览、简化边界 |
| > 60 | 几乎等同于凸包 | 最小外接多边形 |

默认使用 `k = sqrt(N)`（N为点数），自动适配点集规模。

## 算法原理

下面用通俗的语言解释凹包是如何计算出来的。

### 直观理解

想象你有一把图钉撒在桌面上，你想用一根橡皮筋把它们框住。如果直接松手，橡皮筋会收缩成**凸包**——它包住了所有图钉，但可能留出大片空白区域。

凹包则更像是用一根绳子，沿着**最外层**的图钉一个一个绕过去。绳子可以"凹进去"，更紧密地贴合图钉的分布形状。

### 算法步骤

```
输入：二维点集 P，参数 k
输出：凹包边界顶点序列 H

第1步：求凸包
   ┌──────────────────────────┐
   │ 用OpenCV计算点集的凸包    │
   │ 凸包 = 能包住所有点的     │
   │ 最小凸多边形             │
   └──────────────────────────┘
              │
              ▼
第2步：估算点密度
   ┌──────────────────────────┐
   │ 对每个点，找离它最近的点  │
   │ 取所有最近距离的中位数    │
   │ → 得到点间距的"典型值"d  │
   └──────────────────────────┘
              │
              ▼
第3步：计算边长阈值
   ┌──────────────────────────┐
   │ threshold = (k² / 60) × d │
   │ k小 → 阈值小 → 边要短    │
   │ k大 → 阈值大 → 容许多长边 │
   └──────────────────────────┘
              │
              ▼
第4步：迭代细化（核心）
   ┌──────────────────────────────────────┐
   │ 重复以下操作直到所有边都足够短：      │
   │                                      │
   │ ① 遍历当前多边形的每条边             │
   │ ② 找到长度 > 阈值的最长边            │
   │ ③ 在这条边中点附近找最近的未使用的点  │
   │ ④ 将该点插入边的两端点之间           │
   │                                      │
   │ → 就像用剪刀剪开太长的边，             │
   │   在中间缝上一个附近的点              │
   └──────────────────────────────────────┘
              │
              ▼
第5步：输出结果
   ┌──────────────────────────┐
   │ 得到凹包顶点的有序序列    │
   │ 首尾相连形成闭合多边形    │
   └──────────────────────────┘
```
